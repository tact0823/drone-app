# 9. 報告書 PDF テンプレート設計

## 参考資料

**参照元:** `嶋口様邸 屋根・太陽光パネル劣化診断調査報告書`（全 2 部構成、各 5 ページ）

本設計は上記実報告書のレイアウト・項目構成をベースに、**営業支援型報告書** として ThermoInspect Report の MVP 自動生成テンプレートを定義する。

> 評価ロジック詳細: `docs/10-sales-assessment-design.md`

---

## 報告書全体構成

実報告書は **2 部構成**。異常箇所・点検部位が多い場合は Part 2 以降に詳細ページを分割する。

```
Part 1（概要 + 主要所見）
  P1  表紙
  P2  物件概要 + 全景マップ（異常マーカー付き）
  P3  別紙：太陽光パネル熱赤外線診断報告書
  P4  詳細報告（部位別）①
  P5  詳細報告（部位別）②

Part 2（追加所見 + 総括）※ MVP では 1 部 PDF に統合可
  P1  詳細報告（部位別）③
  P2  テキスト総括（調査概要 / 太陽光 / 屋根 / 総合判定）
  P3  点検チェックリスト表
  ...
```

### MVP での生成方針

| 項目 | MVP | Post-MVP |
|------|-----|----------|
| 表紙 | ✅ | |
| 物件概要 + 全景 | ✅ | |
| 太陽光パネル別紙 | ✅（点検種別が SOLAR_PANEL または複合時） | |
| 部位別詳細ページ | ✅（異常 1 件 = 1 ページ） | |
| 総括テキストページ | ✅ | |
| 2 部 PDF 分割 | ❌ | 異常 10 件超で Part 2 自動分割 |
| 可視 + 赤外線ペア | ✅ | |
| 方位コンパス表示 | ✅ | |
| 判定ランク（A/B/C 等） | ✅ | |

### T-04 詳細報告（部位別・1 異常 = 1 ページ）

実報告書の核心テンプレート。異常（anomaly）ごとに 1 ページ生成。

```
┌─────────────────────────────────────────────────┐
│ <報告>                              {page}/{total}│
├─────────────────────────────────────────────────┤
│ 調査年月日 │ {date}    │ 調査者 │ {inspector}  │
│ 調査場所   │ {location}                          │
├─────────────────────────────────────────────────┤
│ 調査部位   │ {partName}  （例: 屋根（スレート））│
│ 診断       │ {diagnosisGrade}                    │
│            │ 例: B — 経年劣化はあるが大きな不具合なし│
├─────────────────────────────────────────────────┤
│ 状況                                           │
│ ・{comment line 1}                              │
│ ・{comment line 2}                              │
├──────────┬──────────────────────────────────────┤
│ 方位     │ [可視画像 — ドローン撮影]              │
│ [コンパス]│                                      │
│          │                                      │
│ 可視画像  │                                      │
│          │                                      │
│ 赤外線画像│ [赤外線画像 — 温度カラースケール付き]  │
│          │                         [温度スケール] │
└──────────┴──────────────────────────────────────┘
```

| フィールド | データソース | MVP |
|-----------|-------------|-----|
| 調査部位 | inspection_type + anomaly.type ラベル | ✅ |
| 診断 | anomalies.overall_grade | ✅ A〜E |
| 状況 | anomalies.comment（箇条書き） | ✅ |
| 方位 | anomalies.direction | ✅ 新規 |
| 可視画像 | ペア画像 VISIBLE | ✅ |
| 赤外線画像 | ペア画像 INFRARED + マーカー矩形 | ✅ |
| ページ番号 | 自動採番 | ✅ |

#### 総合評価スコア（A〜E）

| コード | 表示 | 意味 |
|--------|------|------|
| `A` | A — 正常 | 異常なし |
| `B` | B — 軽度劣化 | 経年劣化あり |
| `C` | C — 要点検 | 計画的対応必要 |
| `D` | D — 要修繕 | 早期修繕推奨 |
| `E` | E — 緊急 | 即時対応必要 |

#### 判定ランク（太陽光別紙用）

| コード | 表示 |
|--------|------|
| `GOOD` | 良好 |
| `CAUTION` | 要観察 |
| `BAD` | 要対応 |

## 点検種別別テンプレート構成

### 太陽光パネル（SOLAR_PANEL）

```
T-01 表紙 → T-02 概要 → T-03 太陽光別紙 → T-04 × N → T-05 総括
```

### 屋根（ROOF）

```
T-01 表紙 → T-02 概要 → T-04 × N → T-05 総括
```

### 外壁（EXTERIOR_WALL）

```
T-01 表紙 → T-02 概要 → T-04 × N → T-05 総括
```

### 屋根 + 太陽光複合（参考 PDF と同型）

```
T-01 表紙（タイトルに「屋根・太陽光パネル」）
  → T-02 概要
  → T-03 太陽光別紙
  → T-04 × N（屋根・太陽光混在）
  → T-05 総括（太陽光 + 屋根セクション）
```

> MVP では `inspection_type` 単一選択。複合点検は Post-MVP で `inspection_types[]` 対応。

## 画像要件

### 画像タイプ

| type | 用途 | 必須 |
|------|------|------|
| `OVERVIEW` | 表紙・全景マップ | 1 枚以上 |
| `VISIBLE` | 可視画像（詳細ページ） | 異常ごと |
| `INFRARED` | 赤外線画像（詳細ページ） | 異常ごと（ペア） |

### 可視 + 赤外線ペアリング

| 方式 | MVP |
|------|-----|
| 同一 `pair_id`（UUID）で VISIBLE + INFRARED を紐付け | ✅ 推奨 |
| 手動ペアリング UI | ✅ |
| EXIF GPS 自動ペアリング | Post-MVP |

### PDF 埋め込み

- 赤外線画像に異常マーカー矩形を SVG/Canvas でオーバーレイしてから PDF 化
- 温度スケールは赤外線画像右側に固定配置（参考 PDF 準拠）
- 方位コンパスは可視画像左側に N/E/S/W アイコン配置

## HTML → PDF 生成方針

### 推奨: Puppeteer

```
1. テンプレート HTML（Handlebars / React SSR）にデータ注入
2. 各ページを A4 サイズ HTML でレンダリング
3. Puppeteer page.pdf() で結合 PDF 生成
4. storage/reports/ に保存
```

### テンプレートファイル構成（Phase 4）

```
backend/src/templates/report/
├── styles/
│   └── report.css          # A4 印刷用スタイル
├── partials/
│   ├── cover.hbs           # T-01
│   ├── overview.hbs        # T-02
│   ├── solar-annex.hbs     # T-03
│   ├── finding-detail.hbs  # T-04
│   ├── summary.hbs         # T-05
│   ├── compass.svg         # 方位コンパス
│   └── temp-scale.svg      # 温度スケール
└── report.hbs              # メインレイアウト
```

### スタイルガイド（参考 PDF 準拠）

| 要素 | 仕様 |
|------|------|
| 用紙 | A4 縦 |
