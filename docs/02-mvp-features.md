# 2. MVP 機能

## MVP の定義

**「Google OAuth でログインし、点検種別を選んでプロジェクトを作成 → 赤外線画像をアップロード → 異常箇所を記録 → AI が診断コメントを生成 → 報告書 PDF をダウンロードできる」**

MVP 外の機能は Post-MVP に回す。

---

## 機能一覧

### ✅ MVP に含める

| # | 機能 | 説明 |
|---|------|------|
| F-01 | Google OAuth ログイン | Google アカウントで認証。初回ログイン時にユーザー自動作成 |
| F-02 | プロジェクト作成 | 点検種別・現場名・日付・場所・メモを登録 |
| F-03 | プロジェクト一覧・詳細 | 自分のプロジェクトを一覧表示・詳細閲覧 |
| F-04 | 画像アップロード | 赤外線 JPEG/PNG を複数枚アップロード |
| F-05 | 画像一覧・プレビュー | アップロード済み画像のサムネイル表示 |
| F-06 | 異常箇所の記録 | 画像上に矩形マーカー + 異常種別 + コメント |
| F-07 | AI 診断コメント自動生成 | 異常保存時に AI が所見・リスク・推奨アクションを自動生成 |
| F-08 | 総合評価スコア | A〜E を異常データから自動算出 |
| F-09 | 異常評価（緊急度・対応時期） | ★1〜5 + 即時/6 か月/1 年/3 年 を自動表示 |
| F-10 | 屋根残寿命推定 | ○〜○ 年を自動推定（ROOF 点検時） |
| F-11 | 太陽光発電リスク | 低・中・高を自動判定（SOLAR_PANEL 点検時） |
| F-12 | 推奨工事プラン | 簡易補修 / 中規模改修 / 大規模改修を自動生成 |
| F-13 | 報告書 PDF 3 種生成 | 調査版 / 顧客提出版 / 営業提案版 |
| F-14 | 報告書ダウンロード | 生成済み PDF のダウンロード |
| F-15 | 管理者：ユーザー一覧 | Admin ロールのみ全ユーザー閲覧（role 表示含む） |

### ❌ MVP に含めない（Post-MVP）

| 機能 | 延期理由 |
|------|----------|
| **Stripe 決済・課金** | MVP は無料提供。決済基盤は Post-MVP |
| AI 自動異常検知（画像解析） | モデル学習・精度検証に時間がかかる（コメント生成は MVP 対象） |
| クライアント向け共有リンク | 認可設計が複雑化する |
| 地図連携（GPS タグ表示） | EXIF 解析・地図 API が追加 scope |
| メール通知 | 外部サービス連携が必要 |
| 組織（テナント）管理 | 単一組織で MVP 開始 |
| 画像の before/after 比較 | UI 複雑度が高い |
| モバイルアプリ | Web レスポンシブのみ |
| メール + パスワード認証 | Google OAuth に統一 |

## AI 診断コメント生成（F-07）

### 概要

異常箇所を記録する際、オペレーターが「AI コメント生成」ボタンを押すと、LLM が診断コメント案を生成する。

### 入力

- 点検種別（`inspection_type`）
- 異常種別（`anomaly.type`）
- 重要度（`severity`）
- オペレーターのメモ（任意）

### 出力

- 報告書にそのまま使える日本語コメント（100〜300 文字）
- オペレーターが編集してから保存

### 制約

- 画像そのものは LLM に送信しない（テキスト情報のみ）
- レートリミット：10 回 / 分 / ユーザー
- API キーはサーバー側のみ保持

---

## ユーザーフロー（MVP）

```
[Google OAuth ログイン]
    ↓
[ダッシュボード] ── プロジェクト一覧
    ↓
[プロジェクト作成] ── 点検種別選択 + 現場名・日付・場所入力
    ↓
[プロジェクト詳細]
    ├── [画像アップロード] ── 複数枚選択 → アップロード
    ├── [画像一覧] ── サムネイル表示
    │       ↓
    │   [異常記録] ── 矩形描画 + 種別選択
    │       ↓
    │   [AI コメント生成] ── コメント案 → 編集 → 保存
    └── [報告書生成] ── PDF 生成 → ダウンロード
```

## 営業支援型評価（MVP 必須）

> 詳細: `docs/10-sales-assessment-design.md`

### 総合評価スコア（自動算出）

| スコア | 名称 |
|--------|------|
| A | 正常 |
| B | 軽度劣化 |
| C | 要点検 |
| D | 要修繕 |
| E | 緊急 |

### 異常ごと表示項目

| 項目 | 値 |
|------|-----|
| 緊急度 | ★1〜★5（自動算出、手動調整可） |
| 推奨対応時期 | 即時 / 6 か月以内 / 1 年以内 / 3 年以内 / 経過観察 |
| AI 診断コメント | 異常保存時に自動生成 |

### プロジェクト評価

| 項目 | 条件 |
|------|------|
| 屋根残寿命 | ROOF 点検時：`○〜○ 年` |
| 太陽光発電リスク | SOLAR_PANEL 点検時：`低 / 中 / 高` |
| 推奨工事プラン | 報告書最終ページ：簡易補修 / 中規模改修 / 大規模改修 |

### 報告書 3 種類

| 種別 | 用途 |
|------|------|
| 調査版（SURVEY） | 社内技術記録 |

## 報告書テンプレート構成

> 詳細: `docs/09-report-template-design.md`（参考 PDF: 嶋口様邸 屋根・太陽光パネル劣化診断調査報告書）

### ページ構成（MVP）

| # | テンプレート | 内容 |
|---|-------------|------|
| T-01 | 表紙 | 依頼者名・タイトル・全景写真・調査日・担当者・会社情報 |
| T-02 | 物件概要 | 物件情報表 + 全景マップ（異常マーカー A/B/C…） |
| T-03 | 太陽光別紙 | 太陽光パネル熱赤外線診断（一覧表 + 可視/赤外線ペア）※ SOLAR_PANEL 時 |
| T-04 | 詳細報告 | 異常 1 件 = 1 ページ（診断グレード + 状況 + 可視/赤外線ペア + 方位） |
| T-05 | 総括 | 調査概要 / 点検種別別結果 / 総合判定 |
| T-06 | 評価サマリー | 総合スコア + 屋根残寿命 + 太陽光リスク ※営業版 |
| T-07 | 推奨工事プラン | 簡易補修 / 中規模改修 / 大規模改修 ※営業版のみ |

### 点検種別別ページ構成

| 点検種別 | ページ構成 |
|----------|-----------|
| SOLAR_PANEL | T-01 → T-02 → T-03 → T-04×N → T-05 → T-06 |
| ROOF | T-01 → T-02 → T-04×N → T-05 → T-06 |
| EXTERIOR_WALL | T-01 → T-02 → T-04×N → T-05 → T-06 |

**営業提案版（SALES）:** 上記 + T-07  
**調査版（SURVEY）:** 上記 + 技術詳細  
**顧客提出版（CUSTOMER）:** T-01〜T-05（T-06/T-07 除外）

### 画像要件

- **OVERVIEW** — 表紙・全景用（1 枚以上）
- **VISIBLE + INFRARED ペア** — 詳細ページ用（異常ごと）
- 方位コンパス（N/E/S/W）+ 温度スケールを PDF に配置

### 総合評価スコア（A〜E）

| コード | 名称 | 旧 diagnosis_grade からの移行 |
|--------|------|-------------------------------|
| A | 正常 | 問題なし |
| B | 軽度劣化 | 経年劣化あり |
| C | 要点検 | 要監視 |
| D | 要修繕 | 要修理 |
| E | 緊急 | （新規） |
