- Google OAuth ボタンのみ（メール/パスワード入力なし）

#### Safari 向けエラーメッセージ

| error コード | 表示メッセージ |
|-------------|---------------|
| `cookie_blocked` | 「Safari のプライバシー設定によりログインできませんでした。Safari の設定 > プライバシー > 「サイト越えトラッキングを防ぐ」をオフにするか、Chrome をお試しください。」 |
| `oauth_denied` | 「Google ログインがキャンセルされました。もう一度お試しください。」 |
| `oauth_failed` | 「ログインに失敗しました。しばらく待ってから再度お試しください。」 |
| `session_expired` | 「セッションの有効期限が切れました。再度ログインしてください。」 |
| `state_mismatch` | 「セキュリティエラーが発生しました。もう一度ログインしてください。」 |

- iOS Safari ではタップ領域を 44px 以上確保（Apple HIG 準拠）
- Google ログインボタンは Safari の `-webkit-appearance` 崩れを防ぐため Tailwind で明示スタイル

### S-02 OAuth コールバック

```
処理フロー（画面表示なし）:
1. Google からリダイレクト
2. バックエンドで JWT Cookie 設定
3. /auth/me でセッション確認
   - 成功 → ダッシュボード (/) へリダイレクト
   - 失敗 → /login?error=cookie_blocked へリダイレクト
```

- Safari で Cookie 設定失敗を検知し、S-01 へエラー付きリダイレクト
- ローディングスピナーを短時間表示（OAuth 処理中）

## レスポンシブ・Safari UI 方針

### ブレークポイント

- **MVP：デスクトップ優先**（1280px 以上を主ターゲット）
- タブレット（768px〜）：レイアウト調整
- iOS Safari：主要操作（ログイン・アップロード・ダウンロード）を検証

### Safari UI 検証項目（TailwindCSS + React）

| 項目 | 検証内容 |
|------|----------|
| Flexbox / Grid | Safari 固有の `-webkit-` プレフィックス不要確認（Tailwind 自動対応） |
| `100vh` 問題 | iOS Safari のアドレスバーによる高さずれ → `min-h-dvh` 使用 |
| `-webkit-appearance` | ボタン・input のデフォルトスタイルを Tailwind で明示上書き |
| `position: sticky` | ヘッダー固定が Safari / iOS Safari で動作するか確認 |
| Canvas（異常マーカー） | Safari で矩形描画・タッチイベントが動作するか確認 |
| フォント | システムフォントスタック（`-apple-system` 含む） |
| タップ領域 | iOS Safari でインタラクティブ要素 44px 以上 |
| スクロール | `-webkit-overflow-scrolling: touch`（必要時） |

### 検証タイミング

- Phase 1 完了時：ログイン画面（Safari / iOS Safari）
- Phase 3 完了時：画像アップロード + Canvas（HEIC 含む）
- Phase 4 完了時：PDF ダウンロード
- Phase 5 完了時：全画面クロスブラウザ確認
